Features

HLS & DASH

Adaptive streaming via hls.js and dash.js. Supports live streams and VOD. Automatic protocol detection by URL extension.

Live Detection

Automatically detects live streams via LEVEL_LOADED (HLS), isDynamic() (DASH). Shows animated LIVE badge and hides irrelevant controls.

Picture-in-Picture

Native browser PiP support. Automatically disabled for YouTube (iframe API restriction). Guards readyState >= 1 before requesting PiP.

Hot-swap Source

Change src binding or call NgxPlayerControlService.load() to swap the source live. Prior state is cleared; volume and mute are preserved.

CSS Theming

CSS custom properties with --ngx-sp-* prefix. Change colors, radius, and control size via the theme input or directly in CSS.

Angular Signals

Fully reactive API with input(), output(), computed() and effect() from Angular 17+. OnPush change detection throughout.

Multi-player

Multiple simultaneous instances via unique playerId. Each instance gets its own isolated PlayerService and PlayerStateService.

Subtitles

Subtitle support for HLS, DASH and native MP4. Automatic track detection. Language selector in the settings menu.

Getting Started

1

Install dependencies

bash
npm install ngx-streaming-player hls.js dashjs
2

Import the component

typescript
import { StreamingPlayerComponent } from 'ngx-streaming-player';

@Component({
  standalone: true,
  imports: [StreamingPlayerComponent],
  // ...
})
export class MyComponent { }

Use it in your template

Three ways to configure - pick the one that fits your use case:

Config object (full control)
typescript
playerConfig: PlayerConfig = {
  src: 'https://example.com/stream.m3u8',
  autoplay: false,
  muted: false,
  volume: 1.0,
  enablePiP: true,
  enableKeyboard: true,
  playbackRates: [0.5, 1, 1.5, 2],
  controlsLayout: { autoHide: true, autoHideDelay: 3000 },
  theme: { primaryColor: '#E76F51' },
};
html
<ngx-sp-player [config]="playerConfig"></ngx-sp-player>
Individual inputs (simple)
html
<ngx-sp-player
  [src]="streamUrl"
  [autoplay]="true"
  [muted]="false"
  [volume]="0.8">
</ngx-sp-player>
Mixed - hot-swap via binding
html
<!-- Changing currentSrc triggers automatic hot-swap -->
<ngx-sp-player
  [config]="baseConfig"
  [src]="currentSrc">
</ngx-sp-player>

Live Streaming

The player auto-detects live streams. When detected, the progress bar and skip buttons disappear, the speed control is hidden, and an animated LIVE badge appears.

Auto-detection method by protocol

HLS - LEVEL_LOADED event with details.live === true.
DASH - isDynamic() on the manifest.
Native MP4 - infinite/very large duration fallback.

Minimal live stream config
typescript
liveConfig: PlayerConfig = {
  src: 'https://live-stream.m3u8',
  // Live is auto-detected - no extra config needed
  autoplay: true,
  muted: true,        // required for autoplay in most browsers
  enablePiP: true,
};
Reading the live state
typescript
@Component({ ... })
export class MyComponent {
  private ctrl = inject(NgxPlayerControlService);

  get isLive(): boolean {
    return this.ctrl.getState()?.isLive() ?? false;
  }

  get liveText(): string {
    return this.isLive ? '🔴 Live' : 'VOD';
  }
}
Speed control automatically hidden for live streams

When isLive() is true, the playback speed option is removed from the settings menu - changing speed has no effect on a live buffer.

Hot-swap Protocol

Switch between HLS, DASH, MP4, and YouTube at runtime without re-creating the component. The Demo Player section above demonstrates this live.

Via NgxPlayerControlService
typescript
@Component({ ... })
export class ChannelSwitcher {
  private ctrl = inject(NgxPlayerControlService);

  switchTo(url: string): void {
    // Destroys current adapter, resets state,
    // preserves volume + mute, starts new adapter
    this.ctrl.load(url, { autoplay: true }, 'my-player');
  }
}
Via reactive binding
typescript
// Changing [src] or [config] triggers hot-swap automatically
currentSrc = signal('https://stream1.m3u8');

switch(url: string): void {
  this.currentSrc.set(url); // hot-swap fires
}
html
<ngx-sp-player
  [src]="currentSrc()"
  [config]="baseConfig"
  playerId="channel-player">
</ngx-sp-player>
What resets on hot-swap

Playback position, buffer, quality levels, live state, subtitles - all reset. What is preserved: volume level and mute state.

  { id: 'cam2', src: 'https://stream2.m3u8', label: 'Camera 2' },
  { id: 'cam3', src: 'https://stream3.m3u8', label: 'Camera 3' },
  { id: 'cam4', src: 'https://stream4.m3u8', label: 'Camera 4' },

Template - dynamic grid
html
<div class="cam-grid">
  @for (cam of cameras; track cam.id) {
    <div class="cam-card">
      <h4>{{ cam.label }}</h4>
      <ngx-sp-player

Control each player by ID
typescript
@Component({ ... })
export class WallComponent {
  private ctrl = inject(NgxPlayerControlService);

  muteAll(): void {
    this.cameras.forEach(c =>
      this.ctrl.setMuted(true, c.id)
    );
  }

  pauseAll(): void {
    this.cameras.forEach(c =>
      this.ctrl.pause(c.id)
    );
  }

  focusOn(id: string): void {
    // Pause all except the focused one
    this.cameras.forEach(c => {
      if (c.id !== id) this.ctrl.pause(c.id);
    });
    this.ctrl.play(id);
  }
}
Performance tip: lazy mounting

For large grids, use an IntersectionObserver to only mount players when they enter the viewport. The Playground already does this - players outside the scroll area are replaced with a lightweight placeholder.

Programmatic Control

NgxPlayerControlService lets you control any player from anywhere in your app - a parent component, a shared service, a keyboard shortcut handler.

Inject and control
typescript
import {
  NgxPlayerControlService
} from 'ngx-streaming-player';

@Component({ ... })
export class VideoController {
  private ctrl = inject(NgxPlayerControlService);

  // Playback control
  play()   { this.ctrl.play();   }
  pause()  { this.ctrl.pause();  }
  toggle() {
    const state = this.ctrl.getState();
    state?.isPlaying()
      ? this.ctrl.pause()
      : this.ctrl.play();
  }

  // Navigation
  jumpTo(seconds: number): void {
    this.ctrl.seek(seconds);
  }
  skipForward(): void {
    const t = this.ctrl.getState()?.currentTime() ?? 0;
    this.ctrl.seek(t + 10);
  }

  // Audio
  mute()   { this.ctrl.setMuted(true);  }
  unmute() { this.ctrl.setMuted(false); }
  volume(v: number) { this.ctrl.setVolume(v); }

  // Quality / Speed
  setHD()  { this.ctrl.setQuality('1080p'); }
  halfSpeed() { this.ctrl.setPlaybackRate(0.5); }
}
Read reactive state
typescript
@Component({
  template: `
    <p>Playing: {{ isPlaying() }}</p>
    <p>Progress: {{ progress() | number:'1.0-0' }}%</p>
    <p>Buffered: {{ buffered() | number:'1.0-0' }}%</p>
    <p>Duration: {{ duration() }}s</p>
    <p>Live: {{ isLive() }}</p>
  `
})
export class PlayerInfo {
  private ctrl = inject(NgxPlayerControlService);
  private state = this.ctrl.getState();

  // All signals - update automatically
  isPlaying  = this.state?.isPlaying;
  progress   = this.state?.progress;
  buffered   = this.state?.bufferedPercentage;
  duration   = this.state?.duration;
  isLive     = this.state?.isLive;
  isBuffering= this.state?.isBuffering;
  volume     = this.state?.volume;
}

Custom Theming

Customize every color, border radius, and control size using the theme input or directly with --ngx-sp-* CSS custom properties.

Using the theme input
typescript
playerConfig: PlayerConfig = {
  src: 'https://example.com/video.m3u8',
  theme: {
    primaryColor:    '#7C3AED', // progress bar, active states
    secondaryColor:  '#06B6D4', // secondary accents
    accentColor:     '#A78BFA', // highlights, ripples
    backgroundColor: '#1e1b4b', // player background
    textColor:       '#EDE9FE', // text & icons
    borderRadius:    '6px',     // --ngx-sp-radius
    controlSize:     '40px',    // button hit area
  },
};
Using CSS custom properties
css

YouTube Integration

Pass any YouTube URL - the player detects it automatically and loads the IFrame API. No API key required.

Auto-detected by URL
typescript
// All of these are auto-detected as YouTube:
// https://www.youtube.com/watch?v=dQw4w9WgXcQ
// https://youtu.be/dQw4w9WgXcQ
// https://www.youtube.com/embed/dQw4w9WgXcQ

ytConfig: PlayerConfig = {
  src: 'https://www.youtube.com/watch?v=dQw4w9WgXcQ',
  autoplay: false,
  muted: false,
};
Force YouTube protocol
typescript
// Use protocol: 'youtube' to force it (e.g. for IDs)
ytConfig: PlayerConfig = {
  src: 'dQw4w9WgXcQ', // video ID only
  protocol: 'youtube',
  autoplay: false,
};
YouTube API limitations

No PiP - the IFrame API prevents browser Picture-in-Picture. The PiP button is hidden automatically.
Quality - quality levels reported by the IFrame API are suggestions, not guarantees.
Autoplay - YouTube enforces muted autoplay in most browser contexts.

Events & State

React to player events via (ready), (stateChange), (error) outputs, or read reactive signals directly via NgxPlayerControlService.

Output events
html
<ngx-sp-player
  [config]="playerConfig"
  playerId="my-player"
  (ready)="onReady()"
  (stateChange)="onState($event)"
  (error)="onError($event)">
</ngx-sp-player>
typescript
onReady(): void {
  console.log('Player ready');
  this.ctrl.play();
}

onState(state: PlayerState): void {
  if (state.isBuffering) {
    this.showSpinner = true;
  }
  if (state.isLive) {
    this.showLiveBadge = true;
  }
}

onError(error: unknown): void {
  console.error('Playback error:', error);
  this.showFallback = true;
}
Reactive state signals
typescript
@Component({
  // All state updates trigger change detection
  // automatically - no manual subscribe() needed
})
export class MyComp {
  private ctrl = inject(NgxPlayerControlService);
  readonly state = this.ctrl.getState('my-player');

  // Use directly in template
  // state?.isPlaying()        → boolean
  // state?.isLive()           → boolean
  // state?.isBuffering()      → boolean
  // state?.currentTime()      → number (seconds)
  // state?.duration()         → number (seconds, Infinity if live)
  // state?.progress()         → number (0–100)
  // state?.bufferedPercentage()→ number (0–100)
  // state?.volume()           → number (0–1)
  // state?.muted()            → boolean
  // state?.playbackRate()     → number
  // state?.currentQuality()   → string
  // state?.availableQualities()→ QualityLevel[]
  // state?.supportsPiP()      → boolean
  // state?.isPiP()            → boolean
  // state?.isFullscreen()     → boolean

  readonly watchProgress = computed(() => {
    const p = this.state?.progress() ?? 0;
    return `${p.toFixed(0)}% watched`;
  });
}

API Reference

@input Component Inputs

Input Type Default Description
config PlayerConfig - Full configuration object. Optional when using individual inputs.
src string - Media URL. Overrides config.src. Changing this input triggers automatic hot-swap.
autoplay boolean false Start playing automatically on load.
muted boolean false Start with audio muted.
volume number 1.0 Initial volume level (0.0 – 1.0).
poster string - Poster image URL shown before playback starts.
theme PlayerTheme - Custom colors and dimensions. Overrides config.theme.
playerId string 'default' Unique player ID. Required only when using multiple instances.

@output Component Outputs

Output Payload Description
ready void Player finished initializing and is ready to play.
stateChange PlayerState Emits the full state snapshot on every change (play, pause, buffering, etc.).
error unknown Fatal playback error from the adapter.

interface PlayerConfig

Property Type Description
src * string Media URL. Auto-detected: .m3u8 → HLS, .mpd → DASH, youtube → YouTube, other → native.
protocol 'hls'|'dash'|'native'|'youtube' Force a protocol, bypassing auto-detection.
autoplay boolean Start playback automatically on load.
muted boolean Start muted (required by some browsers for autoplay).
volume number Initial volume 0.0–1.0.
poster string Image shown before play (when currentTime === 0).
playbackRates number[] Speed options in the menu. E.g. [0.5, 1, 1.5, 2]. Hidden automatically for live streams.
enablePiP boolean Enable Picture-in-Picture. Auto-disabled for YouTube.
enableKeyboard boolean Keyboard shortcuts: Space, ←→ seek, ↑↓ volume, F fullscreen, M mute.
controlsLayout PlayerControlsLayout Controls visibility and auto-hide behavior.
theme PlayerTheme Color and dimension customization.

service NgxPlayerControlService

Injectable into any component or service. Controls any player instance at runtime by its playerId.

Method Signature Description
load() (src, config?, id?) Hot-swap the source. Destroys current adapter, resets state, initializes new one.
play() (id?) Start playback.
pause() (id?) Pause playback.
seek() (time, id?) Jump to the given time in seconds.
setVolume() (vol, id?) Set volume (0.0–1.0).
setMuted() (muted, id?) Mute or unmute audio.
setPlaybackRate() (rate, id?) Change playback speed (ignored on live streams).
setQuality() (quality, id?) Force a quality level (e.g. '720p') or 'auto'.
getState() (id?) Returns the PlayerStateService with all state signals.
isRegistered() (id?) Check if a player is registered and active.

F Toggle fullscreen
